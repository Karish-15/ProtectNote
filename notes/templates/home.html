Users

  • /users/register  [POST]
    Send form-data to create a user 
  • /users/giveuser  [POST]
    Send token in Authorization header to get user info from said token 
  • /token  [POST]
    Send form-data with username and password to get Access and Refresh JWT tokens

Notes (Send JWT Access token)

Following routes require user to be authenticated


  • /notes/create  [POST]
    Send form-data to create a note 
  • /notes/all  [GET]
    Return all notes created by current user 
  • /notes/<slug:uniqueID>  [GET]
    Retrieves note by uniqueID.
    Password needed for protected notes. No need for authentication if public note 
  • /notes/<slug:uniqueID>/edit  [PUT/DELETE]
    Edit the note (provide relevant form-data) or else delete the note 
  • /notes/<slug:uniqueID>/getpassword  [GET]
    Give password for a protected note.
    The note should be created by current user.
